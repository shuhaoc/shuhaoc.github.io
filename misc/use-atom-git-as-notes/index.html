使用Atom+Git构建笔记系统

Posted on 2018-08-17 | In 杂项 | Views:

用过各种“云笔记”作为自己工作学习的笔记系统，遇到过一些问题。

功能上，版本控制，多端同步冲突，编辑历史回溯等；离线编辑。

安全性，公司可能禁止将文档存储在公开渠道的云笔记上。

作为程序员，可以使用我们熟悉的工具自行解决这些问题。笔者推荐使用Atom+Git来构建自己的笔记系统。

Atom是一个开源的文字、代码编辑器，扩展性非常强，编辑体验也不错。默认安装包内置Git插件和Markdown插件。

Git本身就是一个文件系统，能解决版本控制、离线编辑问题，可以用文件夹实现笔记分类。仓库可以托管在公司内网，非敏感的也可以托管在公开仓库中。

所有操作在本地，push/pull操作只有增量，网络上非常高效。本地永远有副本，不用担心云笔记服务可用性问题。

支持Markdown解决了编辑体验问题，借助Atom丰富的功能，实际上可以作为一个非常好用的笔记客户端使用。
